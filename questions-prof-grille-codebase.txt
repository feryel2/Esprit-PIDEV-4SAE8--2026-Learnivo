AA1 - LOGIQUE METIER ET ANALYSE
===============================

1. Quelle est la logique metier avancee de votre projet, au-dela du simple CRUD ?
-----------------------------------------------------------------------------
Reponse attendue
Notre projet ne fait pas seulement du CRUD. Il integre de vraies regles metier :
- progression par chapitres
- deverrouillage du quiz uniquement apres completion des chapitres
- calcul de progression en pourcentage
- gestion de publication des quiz
- archivage automatique d'anciens quiz publies
- evaluation avec score pondere
- hints IA avec fallback local
- synchronisation cours/quiz entre microservices

Explication
Un projet CRUD simple se limite a creer, lire, modifier et supprimer. Ici, il y a des contraintes metier et des comportements automatiques.

Ou montrer le code
- backend/course-service/src/main/java/com/esprit/courseservice/service/CourseService.java
- backend/quiz-service/src/main/java/com/esprit/quizservice/service/QuizService.java
- frontend/src/app/services/learning-progress.service.ts
- frontend/src/app/pages/chapter-detail.component.ts


2. Comment avez-vous implemente la progression pedagogique par chapitres ?
-------------------------------------------------------------------------
Reponse attendue
La progression est geree dans le frontend avec un service dedie. Un chapitre n'est accessible que si le precedent est complete. Le quiz est deverrouille seulement quand tous les chapitres sont termines.

Explication
La logique se trouve dans LearningProgressService avec des methodes comme :
- isChapterUnlocked(...)
- isQuizUnlocked(...)
- markChapterCompleted(...)
- getProgressPercent(...)

Ou montrer le code
- frontend/src/app/services/learning-progress.service.ts
- frontend/src/app/services/learning-progress.service.spec.ts


3. Comment calculez-vous le pourcentage de progression ?
-------------------------------------------------------
Reponse attendue
Le projet separe la progression des chapitres et la progression finale avec quiz.
Les chapitres representent 50 pour cent de la progression, puis le quiz complete la progression jusqu'a 100 pour cent.

Explication
Dans le service, si tous les chapitres sont completes mais que le quiz n'est pas encore reussi, la progression peut etre 80 pour cent en cas d'echec ou 100 pour cent en cas de reussite.

Ou montrer le code
- frontend/src/app/services/learning-progress.service.ts


4. Quelle est la logique metier du module quiz ?
-----------------------------------------------
Reponse attendue
Le module quiz integre plusieurs regles :
- un quiz publie doit avoir une date de publication
- les identifiants des questions doivent etre uniques
- un seul quiz publie actif par cours
- calcul du score selon les poids des questions
- email recapitulatif apres tentative

Explication
Le service QuizService ne se contente pas d'enregistrer un quiz, il controle sa validite et son comportement metier.

Ou montrer le code
- backend/quiz-service/src/main/java/com/esprit/quizservice/service/QuizService.java
- backend/quiz-service/src/test/java/com/esprit/quizservice/service/QuizServiceTest.java

AA2 - ARCHITECTURE LOGICIELLE
=============================

6. Pourquoi avez-vous choisi une architecture microservices ?
-------------------------------------------------------------
Reponse attendue
Nous avons choisi les microservices pour separer les responsabilites :
- user-service pour l'authentification
- course-service pour les cours
- quiz-service pour les quiz
- api-gateway comme point d'entree unique
- discovery-server pour l'enregistrement des services

Explication
Cette separation facilite la maintenance, l'evolution du projet et la communication entre modules.

Ou montrer le code
- backend/README.md
- backend/api-gateway
- backend/discovery-server
- backend/course-service
- backend/quiz-service
- backend/user-service


7. A quoi sert l'API Gateway dans votre projet ?
-----------------------------------------------
Reponse attendue
L'API Gateway est le point d'entree unique pour le frontend. Il centralise :
- le routage vers les bons microservices
- la securite
- la gestion des acces

Explication
Le frontend n'appelle pas directement chaque service, il passe par le gateway.

Ou montrer le code
- backend/api-gateway/src/main/java/com/esprit/apigateway/config/SecurityConfig.java
- backend/README.md


8. Pourquoi OpenFeign est-il utile dans votre projet ?
------------------------------------------------------
Reponse attendue
OpenFeign permet au course-service d'appeler le quiz-service comme s'il appelait une simple methode Java.
Il est utilise pour garder la coherence entre les cours et les quiz.

Exemple
- si le titre d'un cours change, il faut renommer le titre associe dans les quiz
- si un cours est supprime, il faut supprimer les quiz lies

Ou montrer le code
- backend/course-service/src/main/java/com/esprit/courseservice/service/QuizServiceFeignClient.java
- backend/course-service/src/main/java/com/esprit/courseservice/service/QuizServiceClient.java
- backend/course-service/src/main/java/com/esprit/courseservice/service/CourseService.java
- backend/quiz-service/src/main/java/com/esprit/quizservice/controller/QuizController.java

11. Quel est le trajet complet d'un appel OpenFeign dans votre projet ?
----------------------------------------------------------------------
Reponse attendue
Le trajet est :
1. un admin modifie ou supprime un cours
2. CourseService detecte le changement
3. CourseService appelle QuizServiceClient
4. QuizServiceClient appelle QuizServiceFeignClient
5. Feign envoie la requete HTTP au quiz-service
6. QuizController recoit l'appel
7. QuizService met a jour ou supprime les quiz

Ou montrer le code
- backend/course-service/src/main/java/com/esprit/courseservice/service/CourseService.java
- backend/course-service/src/main/java/com/esprit/courseservice/service/QuizServiceClient.java
- backend/course-service/src/main/java/com/esprit/courseservice/service/QuizServiceFeignClient.java
- backend/quiz-service/src/main/java/com/esprit/quizservice/controller/QuizController.java
- backend/quiz-service/src/main/java/com/esprit/quizservice/service/QuizService.java

---------------------------------------------------------
Reponse attendue
La securite repose sur JWT et les roles.

- STUDENT

Les endpoints sensibles sont proteges au niveau du gateway et des services.

13. Pourquoi securiser a la fois le gateway et les microservices ?
-----------------------------------------------------------------
Reponse attendue
Le gateway filtre les acces au niveau global, mais les microservices doivent aussi proteger leurs endpoints pour eviter qu'un service soit appele directement sans controle.

Explication
C'est une defense en profondeur.

Ou montrer le code
- backend/api-gateway/src/main/java/com/esprit/apigateway/config/SecurityConfig.java
- backend/course-service/src/main/java/com/esprit/courseservice/config/SecurityConfig.java
- backend/quiz-service/src/main/java/com/esprit/quizservice/config/SecurityConfig.java

- test plus integre : CourseServiceSecurityTests, QuizServiceSecurityTests, ApiGatewaySecurityTests
